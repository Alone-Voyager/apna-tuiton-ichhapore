## Development Guidelines

### Code Quality
- Write clean, well-documented, and maintainable code
- Follow Company style guides and coding standards
- Write meaningful commit messages
- Include tests for new features and bug fixes

### Pull Request Process
1. Create a feature branch from `main` (or `develop` if applicable)
2. Make your changes and commit with clear messages
3. Push your branch and create a pull request
4. Request review from designated team members (see CODEOWNERS)
5. Address review feedback
6. Do not merge without approval from required reviewers
7. Delete your branch after merge

### Branching Strategy
- `main` (or `master`): Production-ready code (protected)
- `develop`: Integration branch (if applicable)
- `feature/*`: New features
- `bugfix/*`: Bug fixes
- `hotfix/*`: Urgent production fixes
